# Incident Response Playbook (Current)

最終更新: 2026-02-13
文書オーナー: Security Officer
レビュー周期: 四半期
承認者: Security Officer + Product Owner

## 1. 目的

インシデント時の初動、封じ込め、復旧、再発防止を標準化する。

## 2. 優先度定義

- P1: 業務停止、データ損失懸念、認証障害
- P2: 主要機能障害、重大性能劣化
- P3: 軽微障害

## 3. 初動（P1/P2共通）

1. 事象記録（時刻、症状、影響範囲）
2. 影響判定（ユーザー数、データ影響、法令影響）
3. エスカレーション（PO/SO/TL/VL）
4. 変更凍結判断（必要ならREAD_ONLY/MAINTENANCE）

## 4. 封じ込め

- 認証/権限問題: 該当ユーザーやセッションを無効化
- データ整合性問題: 書き込み停止、バックアップ保全
- 外部連携問題: 連携停止またはキュー停止

## 5. 復旧

- 暫定回避策を提供
- 恒久対策をチケット化
- 変更管理フローで修正を本番適用
- 必要に応じてバックアップから復旧

## 6. 事後対応

- 24時間以内: 一次報告
- 72時間以内: 根本原因、恒久対策、文書更新
- 月次: 再発防止レビュー
