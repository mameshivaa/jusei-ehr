# OSS Distribution Boundary

## Purpose

公開リポジトリに含める範囲と、公開ビルドで無効化する範囲を定義します。

## Included in OSS

- 患者管理、カルテ、来院、施術録、入出力、バックアップなどの主要機能
- ローカルDB上の監査ログ・運用ログ
- ビルドスクリプト、デスクトップ実行スクリプト
- ドキュメント、テスト、開発ツール

## Disabled in OSS

- 外部サービスへの自動データ送信
- 初期設定時の外部向け疎通処理
- 定期実行ジョブによる外向き通信
- 外部識別子の自動発行・送信処理

## Guardrails

- 公開ビルドでは上記機能が有効にならないよう、実行時ガードを維持する
- 初期設定・設定画面に外部送信の選択肢を表示しない
- デフォルト構成で外向き通信が発生しないことを確認する

## Release checklist (OSS repo)

1. `.env.example` に実値の秘密情報・送信先URLが含まれていないこと
2. 初期設定フローに外部送信設定のUIが表示されないこと
3. `npm run type-check` と `npm run lint` が通ること
4. 公開リポジトリへソースを反映すること
