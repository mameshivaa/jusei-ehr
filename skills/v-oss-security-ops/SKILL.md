# V-OSS Security Ops

## Overview

セキュリティインシデント対応、監査ログ確認、運用・法対応文書の参照を一貫した流れで行う。

## Workflow

1. 事象の把握と影響範囲の確認
2. インシデントレベルの分類
3. 迅速な封じ込め/停止と証跡保全
4. 監査ログ・システムログの確認
5. 関係者/窓口への連絡
6. 復旧と再発防止の整理
7. 必要な文書更新

## Notes

- 初動手順やレベル分類は `docs/current/INCIDENT_RESPONSE_PLAYBOOK.md` を優先
- 技術的対策の根拠は `docs/current/SECURITY_BASELINE.md` を参照
- 文書更新が必要な場合は `docs/current/DOCUMENT_CONTROL_POLICY.md` の規程を守る
